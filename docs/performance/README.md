# Performance Benchmarks

This directory contains published performance benchmark reports for the faultsense agent.

## Files

- **`current.md`** — The latest benchmark report, generated by `npm run benchmark:demo` against `examples/todolist-htmx` in at-rest mode (no `fs-*` attributes active).
- **`current.json`** — Machine-readable companion to `current.md` with full per-run measurements.

## How to reproduce

```bash
# From the repo root:
npm run build:agent
npm run benchmark:demo
```

This boots the HTMX demo app locally, runs 10 paired A/B measurements across two throttle profiles (unthrottled + Slow 4G), and writes the results to this directory.

## Interpreting the report

The report compares page performance **without** the faultsense agent (condition A) vs **with** it (condition B). Key sections:

- **Headline summary** — one-line verdict with the most important deltas
- **Environment** — exact hardware, software versions, and run parameters for reproducibility
- **Results tables** — per-profile breakdowns with median, p95, IQR, and significance labels
- **Methodology** — how measurements were taken, what the soak window captures
- **Caveats** — what this report does and does not measure

## Significance labels

- **within noise** — delta is below `max(baseline IQR, 5% of baseline median)`
- **measurable** — delta is between 1x and 2x the noise threshold
- **significant** — delta exceeds 2x the noise threshold

## Trusted-runner path

For procurement or security review: run the tool yourself on your own hardware against your own URL. The strongest answer to "prove it doesn't hurt my site" is numbers you generated.

```bash
npm run benchmark -- https://your-site.com
```

See [`tools/benchmark/README.md`](../../tools/benchmark/README.md) for full usage.
