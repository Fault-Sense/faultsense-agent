# Performance Benchmarks

This directory contains published performance benchmark reports for the faultsense agent.

## Files

- **`current.md`** — The latest demo benchmark report, generated by `npm run benchmark:demo` against `examples/todolist-htmx` with idle soak and active interaction scenarios.
- **`current.json`** — Machine-readable companion to `current.md` with full per-run measurements.
- **`stress.md`** — Stress benchmark scaling curve across 50–1,000 concurrent assertions with background DOM churn.
- **`stress.json`** — Machine-readable companion to `stress.md`.
- **`analysis.md`** — Performance analysis synthesizing both benchmark suites with key findings, scaling curves, and measurement validation.

## How to reproduce

```bash
# From the repo root:
npm run build:agent

# Demo benchmark (30 pairs, ~2 hours)
npm run benchmark:demo

# Stress benchmark (scaling curve, ~4 hours)
cd tools/benchmark/stress && npm install && cd ../../..
npm run benchmark:stress

# A-vs-A measurement validation
npm run benchmark:ava
```

The demo benchmark boots the HTMX demo app locally, runs 30 paired A/B measurements across two throttle profiles (unthrottled + Slow 4G) with both idle soak and active interaction scenarios, and writes the results to this directory.

## Interpreting the report

The report compares page performance **without** the faultsense agent (condition A) vs **with** it (condition B). Key sections:

- **Headline summary** — one-line verdict with the most important deltas
- **Environment** — exact hardware, software versions, and run parameters for reproducibility
- **Results tables** — per-profile breakdowns with median, p95, IQR, and significance labels
- **Methodology** — how measurements were taken, what the soak window captures
- **Caveats** — what this report does and does not measure

## Significance labels

- **within noise** — p ≥ 0.05
- **measurable** — 0.01 ≤ p < 0.05
- **significant** — p < 0.01

Significance is assessed via the Wilcoxon signed-rank test (two-tailed, non-parametric) on paired A-B differences. Confidence intervals use the Hodges-Lehmann estimator over Walsh averages.

## Trusted-runner path

For procurement or security review: run the tool yourself on your own hardware against your own URL. The strongest answer to "prove it doesn't hurt my site" is numbers you generated.

```bash
npm run benchmark -- https://your-site.com
```
